# Change: 公共查询能力整合（无登录、本地存储、多记录列表、UI 对齐）

## Why
- 先前分阶段的公共查询改动（基础查询、本地存储、UI 对齐、多记录列表）已全部落地，需统一成单一变更便于追踪。
- 确保公共查询在安全隔离（无登录、无 D1）前提下，同时具备多 Key 列表、自动预填/自动查询和私有页一致的体验。

## What Changes
- 保持公共查询无登录、无 D1 持久化，使用 Worker 代理 Factory AI 接口。
- localStorage 仅存公共 Key 列表（含最近查询结果），支持自动预填与自动查询。
- 公共页面样式与私有仪表盘一致，列表支持多条记录、单行刷新/删除，重复 Key 只覆盖不追加。
- 与私有 Key 管理/自动刷新完全隔离，不影响私有缓存与会话。

## Impact
- 受影响能力：公共 Key 查询/展示（`public-key-usage`）。
- 受影响代码：`src/index.ts` 公共页面与接口（已实现）；新变更统一描述前述三次改动。
